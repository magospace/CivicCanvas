# CivicCanvas Design Standard

## Product Context

MCP-powered Texas public-data dashboard explorer with governed dataset catalog, deterministic prompt parsing, safe bounded queries, source attribution, local saved canvases, and preview-only Miro exports.

## Design Intent

The visual language should support this goal: Build a production Texas public-data canvas for governed dashboards, safe dataset discovery, bounded queries, source attribution, MCP workflows, local persistence, and deploy-ready civic data exploration.

## Current Status

This is an initial portfolio-seeded `DESIGN.md`. Before broad UI or brand work, inspect existing screens, assets, and audience expectations, then refine this file into a product-specific design contract.

## Principles

- Preserve this product's own brand and audience fit.
- Use a quiet, usable product interface before decorative flourishes.
- Keep text readable and avoid overlapping UI.
- Use stable dimensions for fixed controls, toolbars, cards, tables, boards, and media frames.
- Avoid one-note palettes and avoid copying another portfolio app's visual identity.
- Use real product/content imagery when the user needs to inspect the actual product, place, object, or state.
